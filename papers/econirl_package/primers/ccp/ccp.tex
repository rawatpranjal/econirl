\section{Purpose}

CCP, the conditional choice probability estimator, is the structural dynamic
discrete choice estimator that replaces repeated Bellman fixed-point solves
with first-stage policy inversion. Instead of solving the dynamic programming
problem for every candidate payoff vector, CCP first estimates
\(\hat\pi(a\mid s)\) from observed choices and uses the Hotz-Miller inversion
to recover continuation values. Aguirregabiria and Mira's NPL algorithm then
iterates this mapping until it approximates the MLE fixed point.

This document is a known-truth validation tutorial. It does not use real data.
The synthetic DGP supplies the true reward, transition law, value function,
policy, and Type A/B/C counterfactual oracle objects. The validation question
is whether the CCP/NPL implementation recovers these known objects on a
medium-scale finite panel.

\section{Source Papers}

The primary source is Hotz and Miller (1993), which showed that conditional
choice probabilities can invert the dynamic choice problem under additive
separability, conditional independence, and type-I extreme-value shocks.
Aguirregabiria and Mira (2002) introduced nested pseudo-likelihood, or NPL:
given a policy, estimate payoffs; given payoffs, update the policy; repeat.
Their sequence nests the one-step Hotz-Miller estimator at \(K=1\) and the
NFXP/MLE fixed point in the limit. Magnac and Thesmar (2002) are the
identification warning: the discount factor, scale, and reward location need
normalization or exclusion restrictions.

\section{When to Use CCP}

Use CCP when the state and action spaces are tabular, transitions are known or
first-stage estimated, empirical action support is strong, and a full NFXP
inner solve at every outer step is too expensive. CCP is especially useful as a
structural workhorse when a nonparametric first-stage policy can be estimated
well.

Do not use CCP as a recovery benchmark when many states are rarely observed or
one action is almost never chosen. The first-stage empirical CCPs are the
estimator's foundation; if they are poorly supported, the Hotz-Miller inversion
amplifies the noise.

\section{Model}

The observed data are state-action-next-state trajectories
\[
\{(s_{it}, a_{it}, s_{i,t+1}) : i=1,\ldots,N,\ t=1,\ldots,T\}.
\]

Flow payoff is linear in known features:
\[
u_\theta(s,a) = \phi(s,a)^\top \theta.
\]
Transitions are
\[
P_a(s,s') = \Pr(s_{t+1}=s' \mid s_t=s, a_t=a),
\]
the discount factor is \(\beta\), and the type-I extreme-value shock scale is
\(\sigma\).

Given a policy \(\pi\), define the policy-weighted transition matrix
\[
F_\pi(s,s') = \sum_a \pi(a\mid s)P_a(s,s')
\]
and the logit emax correction
\[
e_\pi(s,a) = \gamma_{\mathrm{Euler}} - \log \pi(a\mid s).
\]

Hotz-Miller inversion writes the integrated value under policy \(\pi\) as
\[
\bar V_\pi
=
(I-\beta F_\pi)^{-1}
\sum_a \pi(a\mid s)\{u_\theta(s,a)+e_\pi(s,a)\}.
\]
For linear rewards, this separates into augmented features:
\[
\bar V_\pi(s)
=
W_\phi(s)^\top\theta + W_e(s),
\]
where
\[
W_\phi = (I-\beta F_\pi)^{-1}\sum_a \pi(a\mid s)\phi(s,a),
\qquad
W_e = (I-\beta F_\pi)^{-1}\sum_a \pi(a\mid s)e_\pi(s,a).
\]
The choice-specific value used in the pseudo-likelihood is
\[
\tilde Q_\theta(s,a;\pi)
=
\phi(s,a)^\top\theta
+ \beta \sum_{s'}P_a(s,s')\{W_\phi(s')^\top\theta+W_e(s')\}.
\]

The \(K\)-stage CCP/NPL estimator is
\[
\hat\theta_K
=
\argmax_\theta
\sum_{i,t}
\log
\frac{\exp(\tilde Q_\theta(s_{it},a_{it};\hat\pi^{K-1})/\sigma)}
{\sum_b\exp(\tilde Q_\theta(s_{it},b;\hat\pi^{K-1})/\sigma)}.
\]

\section{Estimator Mechanics}

The implementation follows five steps.

\begin{enumerate}
\item Estimate empirical CCPs with Laplace smoothing:
\[
\hat\pi_0(a\mid s)
=
\frac{N(s,a)+\delta}{N(s)+|\mathcal A|\delta}.
\]
\item Compute \(F_{\hat\pi}\), \(e_{\hat\pi}\), \(W_\phi\), and \(W_e\) once
per NPL step.
\item Build augmented features and maximize a standard multinomial logit
pseudo-likelihood in \(\theta\).
\item Update CCPs from the fitted choice-specific values.
\item Repeat for \(K\) NPL iterations, then compute standard errors from the
full Bellman-constrained likelihood Hessian.
\end{enumerate}

The important implementation detail is the cost shift. NFXP solves a Bellman
fixed point inside every likelihood evaluation. CCP pays one matrix inversion
per NPL step, then each optimizer step is just a logit gradient on augmented
features. In EconIRL, the augmented-feature logit kernel is JIT-compiled once
and reused across NPL steps.

\section{Identification Checks}

CCP inherits the same payoff identification restrictions as NFXP and adds
first-stage support requirements.

\begin{enumerate}
\item \textbf{Reward rank.} The action-dependent feature matrix must have full
column rank after dropping zero rows.
\item \textbf{Action support.} Every action must have enough empirical mass.
Near-zero CCPs make \(e_\pi(s,a)=\gamma-\log \pi(a\mid s)\) unstable.
\item \textbf{State coverage.} The policy is nonparametrically estimated by
state. Sparse states should be pooled, smoothed, or excluded from a main
validation claim.
\item \textbf{Normalization.} The reward scale is fixed at \(\sigma=1\), and
the synthetic DGP uses an exit action and absorbing state as a zero-reward
anchor.
\item \textbf{Transition separation.} This tutorial supplies the true
transition tensor, so the validation isolates the CCP payoff estimator rather
than transition-estimation error.
\end{enumerate}

\section{Pre-Estimation Diagnostics}

Run diagnostics before fitting. For CCP, support diagnostics are not optional:
the estimator starts from empirical CCPs.

\begin{lstlisting}
from experiments.known_truth import (
    SimulationConfig,
    build_known_truth_dgp,
    get_cell,
    run_pre_estimation_diagnostics,
    simulate_known_truth_panel,
)

cell = get_cell("canonical_low_action")
dgp = build_known_truth_dgp(cell.dgp_config)
panel = simulate_known_truth_panel(
    dgp,
    SimulationConfig(n_individuals=2000, n_periods=80, seed=42),
)
diagnostics = run_pre_estimation_diagnostics(dgp, panel)

assert diagnostics.passed
assert diagnostics.feature_rank == diagnostics.num_features
assert diagnostics.is_action_dependent
assert diagnostics.state_action_coverage == 1.0
\end{lstlisting}

\section{Run the Tutorial}

From the repository root:

\begin{lstlisting}
PYTHONPATH=src:. python -m experiments.known_truth \
    --estimator CCP \
    --cell-id canonical_low_action \
    --output-dir outputs/known_truth \
    --show-progress \
    --verbose
\end{lstlisting}

The command builds the DGP, simulates the panel with \texttt{tqdm} progress,
runs pre-estimation diagnostics, fits K=10 CCP/NPL with finite standard
errors, evaluates reward/value/Q/policy recovery, solves Type A/B/C
counterfactual oracles, writes one \texttt{result.json}, and raises if any
non-smoke hard gate fails.

Direct estimator usage:

\begin{lstlisting}
from experiments.known_truth import (
    build_known_truth_dgp,
    get_cell,
    simulate_known_truth_panel,
)
from econirl.estimation.ccp import CCPEstimator

cell = get_cell("canonical_low_action")
dgp = build_known_truth_dgp(cell.dgp_config)
panel = simulate_known_truth_panel(dgp, cell.simulation_config)

estimator = CCPEstimator(
    num_policy_iterations=10,
    ccp_min_count=1,
    ccp_smoothing=1e-6,
    outer_tol=1e-6,
    outer_max_iter=500,
    se_method="robust",
    compute_hessian=True,
    verbose=True,
)

summary = estimator.estimate(
    panel=panel,
    utility=dgp.utility(),
    problem=dgp.problem,
    transitions=dgp.transitions,
)
\end{lstlisting}

\section{Debugging Gaps}

If CCP fails to recover truth, debug in this order.

\begin{enumerate}
\item \textbf{First-stage support.} Check observed states, state-action
coverage, minimum action share, and minimum positive CCP.
\item \textbf{One-action states.} If a state has only one observed action,
Hotz-Miller inversion is extrapolating that state's counterfactual action
values.
\item \textbf{Smoothing.} Too little smoothing makes logs unstable; too much
smoothing biases the first-stage policy toward uniform choice.
\item \textbf{Feature rank.} If rank is below the number of reward parameters,
\(\theta\) is not identified.
\item \textbf{Normalization.} If the exit/absorbing anchor is invalid, reward
levels can drift.
\item \textbf{NPL iterations.} A K=1 one-step estimator can be consistent, but
the validation claim uses K=10 and hard gates.
\item \textbf{Value convention.} CCP inversion includes the Euler-constant
emax correction internally. EconIRL reports values in the package's soft
Bellman convention by evaluating the recovered policy under the recovered
reward, so known-truth value comparisons are meaningful.
\item \textbf{Soft gates.} Do not accept a non-smoke run that only warns.
Known-truth validation should pass hard gates or raise.
\end{enumerate}

\section{Practical Defaults}

Recommended non-smoke validation settings:

\begin{lstlisting}
CCPEstimator(
    num_policy_iterations=10,
    ccp_min_count=1,
    ccp_smoothing=1e-6,
    convergence_tol=1e-6,
    outer_tol=1e-6,
    outer_max_iter=500,
    se_method="robust",
    compute_hessian=True,
)
\end{lstlisting}

Use smoke settings only for code-path tests. A smoke test shows that the
estimator runs and returns finite objects; it is not evidence of recovery.
