\section{Purpose}

NNES estimates dynamic discrete choice models with a neural approximation to
the integrated value function. The motivation is the same structural target as
NFXP, CCP, MPEC, and SEES: recover reward parameters, the implied policy, the
value function, and counterfactual policies. The difference is that NNES uses a
neural value function inside an NPL-style policy iteration step, so it can in
principle scale beyond low-dimensional tabular state spaces.

This document is a known-truth validation tutorial. It does not use real data.
The synthetic DGP supplies the true reward, transition law, value function,
policy, and Type A/B/C counterfactual oracle objects. The package
implementation runs on this DGP and must pass the hard structural recovery
gates before the generated results are written.

\section{Source Paper}

The source paper is \citet{nguyen2025}. The paper embeds a neural value
function approximation in the Nested Pseudo-Likelihood framework of
\citet{aguirregabiriaMira2002}. The key theoretical result is a zero-Jacobian
or Neyman-orthogonality property: at the true conditional choice
probabilities, first-order value-network approximation error drops out of the
structural score.

Under the paper's regularity conditions, if the neural value approximation
converges faster than \(n^{-1/4}\), the structural parameter estimator is
\(\sqrt n\)-consistent and efficient. The orthogonality result is powerful,
but it is not a finite-sample guarantee. The implementation still has to put
the neural value function, policy update, reward parameters, and Bellman level
on a mutually consistent point of the model.

\section{When to Use NNES}

Use NNES when the model is structural, transitions are known or first-stage
estimated, the reward is parametric, and the state space is large enough that
exact tabular value iteration or full MPEC constraints are impractical. NNES is
not a behavioral cloning shortcut: matching observed choices is insufficient
unless the recovered reward and counterfactual policies also match the
structural model.

Use the generated results below as the package-level recovery check for the
finite-state known-truth benchmark. The shared harness checks more than the
training loss or in-sample choice fit: it verifies parameter, reward, policy,
value, Q, and counterfactual recovery.

\section{Model}

\commonnotation

The observed data are state-action-next-state trajectories
\[
\{(s_{it}, a_{it}, s_{i,t+1}) : i=1,\ldots,N,\ t=1,\ldots,T\}.
\]
Flow payoff is linear in known features,
\[
u_\theta(s,a)=\phi(s,a)^\top\theta,
\]
with transition law \(P_a(s,s')\), discount factor \(\beta\), and type-I
extreme-value scale \(\sigma\). Given a continuation value \(V\),
\[
Q_\theta(s,a;V)=u_\theta(s,a)+\beta\sum_{s'}P_a(s,s')V(s'),
\]
and the implied logit policy is
\[
\pi_{\theta,V}(a\mid s)
=
\frac{\exp(Q_\theta(s,a;V)/\sigma)}
{\sum_b \exp(Q_\theta(s,b;V)/\sigma)}.
\]

\section{NNES Estimator}

Given conditional choice probabilities \(P(a\mid s)\), the finite-state NPL
evaluation step profiles the value representation as
\[
W_\theta[P]=W_z[P]\theta+W_e[P],
\]
where \(W_z[P]\) is the expected discounted feature valuation and \(W_e[P]\)
is the expected discounted logit entropy term under the fixed CCP iterate.
The implementation trains a ReLU MLP \(V_\gamma(s)\) by supervised regression
on \(W_\theta[P]\) as the NNES value approximation, then optimizes the
structural parameters through the profiled one-step NPL likelihood rather than
freezing the continuation value at the previous \(\theta\). The reported
validation value function is re-evaluated from the final recovered reward and
policy on the Bellman scale.

\begin{algorithm}[H]
\DontPrintSemicolon
\caption{NNES with NPL value regression}
\KwIn{Panel data, reward features, transitions, discount \(\beta\), initial CCPs, anchor state}
Estimate initial CCPs from empirical state-action frequencies\;
\For{outer iteration \(k=1,\ldots,K\)}{
  Compute the NPL value target \(W_\theta[P_{k-1}]\)\;
  Normalize the target by the anchor state\;
  Train \(V_\gamma\) to predict the anchored target over all states\;
  Maximize the profiled pseudo-likelihood over \(\theta\) through \(W_z[P]\theta+W_e[P]\)\;
  Update CCPs from the implied logit policy\;
}
Evaluate the final policy value on the Bellman scale\;
\KwOut{\(\hat\theta\), value function, policy, log likelihood, diagnostics}
\end{algorithm}

\paragraph{Comparison with SEES.}
SEES \citep{luoSang2024} approximates \(V\) with deterministic basis functions
and penalizes the Bellman residual in a joint optimization problem. NNES uses a
neural value approximation and NPL orthogonality instead. In principle, this
avoids the Schur-complement information matrix needed by SEES. In the
finite-state validation harness, the profiled NPL step gives an exact
policy-evaluation specialization of the paper's neural approximation idea.

\section{Interpretation}

The validation run confirms that the NNES implementation recovers the same
structural objects checked for the other validated estimators: reward
parameters, policy, Bellman-level values, Q values, and counterfactual
policies. The key implementation detail is that the structural MLE step
profiles policy evaluation through \(\theta\), matching the source paper's
one-step NPL construction.
