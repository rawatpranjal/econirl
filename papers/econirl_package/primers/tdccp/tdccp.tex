\section{Purpose}

TD-CCP estimates dynamic discrete choice models with the conditional choice
probability likelihood, but replaces transition-density integration with
temporal-difference regressions. The estimator first estimates CCPs, then uses
observed transition tuples \((a_t,x_t,a_{t+1},x_{t+1})\) to estimate the
recursive objects \(h(a,x)\) and \(g(a,x)\). These objects enter the PMLE
criterion for the structural parameters. No transition matrix is used in that
parameter-estimation stage.

This document is now a known-truth validation tutorial. It uses the shared
synthetic DGP harness in \texttt{experiments/known\_truth.py}; it does not use
real data and it does not reuse the old continuous-state primer. The harness
supplies true rewards, values, policies, and Type A/B/C counterfactual oracles.
The validation question is whether TD-CCP recovers those known objects from
finite panel data.

\section{Source Paper}

The source paper is \citet{adusumilliEckardt2025}. For a known feature vector
\(z(a,x)\) and parameter vector \(\theta\), the paper writes the CCP likelihood
using
\[
\pi(a\mid x;\theta,h,g)
=
\frac{\exp\{h(a,x)^\top\theta + g(a,x)\}}
{\sum_{a'}\exp\{h(a',x)^\top\theta + g(a',x)\}}.
\]
The recursive terms solve
\[
h(a,x)=z(a,x)+\beta E[h(a',x')\mid a,x],
\]
\[
g(a,x)=\beta E[e(a',x')+g(a',x')\mid a,x],
\quad
e(a,x)=\gamma_E-\log P(a\mid x).
\]
The semi-gradient estimator replaces these expectations by sample averages
over observed current and next state-action pairs. In linear form it solves
\[
\hat\omega
=
\left[
E_n\{\phi(a,x)(\phi(a,x)-\beta\phi(a',x'))^\top\}
\right]^{-1}
E_n\{\phi(a,x)z(a,x)\},
\]
with an analogous equation for \(g\) using the target
\(\beta e(a',x')\). This is the point of the implementation: estimate the
recursive terms directly from sample transitions, not from a modeled
transition density.

In package validation, this means TD-CCP is treated as a finite-\(\theta\)
estimator: rewards are linear in known features, while the state features
themselves may be flexible, including frozen neural encodings. It is not used
as a raw neural reward recovery method.

\section{Implementation Notes}

The implementation lives in \texttt{src/econirl/estimation/td\_ccp.py}. The
semi-gradient path supports scalar polynomial bases, encoded-state bases, and
tabular bases. The known-truth migration uses the encoded-state basis so the
same code path can handle high-dimensional state encoders. Reward features can
be included in that basis when they contain nonlinear transformations of the
state encoding.

Known transitions are passed to the estimator only because the package result
object includes a final policy and value function. After \(\hat\theta\) is
estimated, the validation harness solves the dynamic program under the known
synthetic transition law to compare recovered policies, values, and
counterfactuals. That final evaluation does not feed back into the PMLE for
\(\theta\).

\section{Usage}

The high-level wrapper is intended for package users:

\begin{lstlisting}[basicstyle=\footnotesize\ttfamily]
import pandas as pd
from econirl.estimators import TDCCP

data = pd.read_csv("zurcher_bus.csv")

model = TDCCP(
    n_states=90,
    n_actions=2,
    discount=0.9999,
    utility="linear_cost",
    method="semigradient",
    basis_type="encoded",
    basis_dim=3,
)
model.fit(data, state="mileage_bin", action="replaced", id="bus_id")

print(model.params_)
print(model.summary())
\end{lstlisting}
