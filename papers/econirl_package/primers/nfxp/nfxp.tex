\section{Purpose}

NFXP, the nested fixed point estimator, is the reference estimator for structural
dynamic discrete choice. It estimates primitive reward parameters by solving the
agent's dynamic programming problem inside the likelihood. The method is the
baseline against which CCP, MPEC, NNES, TD-CCP, SEES, and IRL-style estimators
should be compared.

This document is a known-truth validation tutorial. It does not use real data.
The synthetic DGP supplies the true reward, transition law, value function,
policy, and counterfactual oracle objects. That lets us ask a sharp question:
when the NFXP assumptions hold, does the implementation recover the ground
truth at medium scale?

\section{Source Papers}

The primary source is Rust (1987), which formalized the bus-engine replacement
problem and introduced nested fixed point maximum likelihood. The computational
upgrade is Iskhakov, Rust, Schjerning, and Seo (2016), which recommends a
successive-approximation plus Newton-Kantorovich inner loop. The identification
lesson that matters for this validation is standard in dynamic discrete choice:
the reward scale and location must be normalized, transitions must be handled
separately from payoffs, and reward features must vary enough across actions to
identify the structural parameters.

\section{When to Use NFXP}

Use NFXP when the state space is tabular, the transition kernel is known or can
be estimated in a first stage, the reward is parameterized by a low-dimensional
vector, and exact dynamic programming is computationally feasible. NFXP is the
right tool when the goal is publication-grade structural estimates, likelihood
values, analytical scores, standard errors, and counterfactual policies from
recovered primitives.

NFXP is usually the wrong tool for high-dimensional or continuous state spaces.
For those cases, use NNES, SEES, GLADIUS, or TD-CCP depending on whether the
goal is structural parameter recovery, value-function approximation, or
transition-free estimation.

\section{Model}

The observed data are state-action-next-state trajectories
\[
\{(s_{it}, a_{it}, s_{i,t+1}) : i=1,\ldots,N,\ t=1,\ldots,T\}.
\]

The flow payoff is linear in known features:
\[
u_\theta(s,a) = \phi(s,a)^\top \theta.
\]
The transition kernel is
\[
P_a(s,s') = \Pr(s_{t+1}=s' \mid s_t=s, a_t=a).
\]
The discount factor is $\beta$ and the i.i.d. type-I extreme-value shock scale
is $\sigma$. The integrated value function solves the soft Bellman fixed point
\[
V_\theta(s)
= \sigma \log \sum_a
\exp\left(
\frac{
u_\theta(s,a)
+ \beta \sum_{s'} P_a(s,s') V_\theta(s')
}{\sigma}
\right).
\]
The choice-specific value is
\[
Q_\theta(s,a)
= u_\theta(s,a) + \beta \sum_{s'} P_a(s,s')V_\theta(s'),
\]
and the implied conditional choice probability is
\[
\pi_\theta(a \mid s)
=
\frac{\exp(Q_\theta(s,a)/\sigma)}
{\sum_b \exp(Q_\theta(s,b)/\sigma)}.
\]

NFXP maximizes the conditional log-likelihood
\[
\hat{\theta}
=
\argmax_\theta
\sum_{i,t} \log \pi_\theta(a_{it} \mid s_{it}).
\]

\section{Estimator Mechanics}

For each candidate $\theta$, the inner loop solves $V=TV$. EconIRL supports
successive approximation, Newton-Kantorovich, and a hybrid solver. The default
known-truth validation uses the hybrid solver: safe successive approximation
far from the fixed point and Newton-Kantorovich updates near the solution.

The analytical score differentiates through the fixed point. For observation
$i$, the score has the logit form
\[
\psi_i(\theta)
=
\frac{1}{\sigma}
\left[
\nabla_\theta Q(s_i,a_i)
-
\sum_a \pi_\theta(a\mid s_i)\nabla_\theta Q(s_i,a)
\right],
\]
with
\[
\nabla_\theta Q(s,a)
=
\phi(s,a)
+
\beta \sum_{s'} P_a(s,s')\frac{\partial V(s')}{\partial \theta}.
\]
The total derivative solves
\[
(I-\beta P_\pi)\frac{\partial V}{\partial \theta}
=
\sum_a \pi_\theta(a\mid s)\phi(s,a).
\]

The outer loop uses BHHH by default. It builds the positive semi-definite
outer-product matrix from per-observation scores, applies damping when needed,
and uses a line search to maintain likelihood improvement.

\section{Identification Checks}

The known-truth DGP must be identified before we judge the estimator.
Rust-style NFXP handles this through three restrictions.

\begin{enumerate}
\item \textbf{Conditional independence.} The state transition is Markov in the
observed state and action and does not depend on the current logit shock.
\item \textbf{Transition separation.} The transition process is not estimated
inside the payoff likelihood in this tutorial. The synthetic DGP supplies the
true transition tensor exactly.
\item \textbf{Reward normalization.} The shock scale is fixed at $\sigma=1$.
The synthetic DGP uses an exit action and absorbing state as a zero-reward
anchor, so payoff location is pinned down.
\end{enumerate}

The practical identification failure to watch for is action support. If one
action is almost never chosen, the likelihood cannot pin down that action's
payoff. The second failure is state-only reward variation copied across
actions. That collapses action-specific payoff differences and makes a
structural theta recovery test ill posed.

\section{Run the Tutorial}

From the repository root:

\begin{lstlisting}
PYTHONPATH=src:. python -m experiments.known_truth \
    --estimator NFXP \
    --cell-id canonical_low_action \
    --output-dir outputs/known_truth \
    --show-progress \
    --verbose
\end{lstlisting}

The command builds the DGP, solves the true Bellman problem, simulates the
panel with \texttt{tqdm} progress, runs pre-estimation diagnostics, fits NFXP,
evaluates reward/value/Q/policy recovery, solves Type A/B/C counterfactual
oracles, writes one \texttt{result.json}, and raises if any non-smoke hard gate
fails. NFXP itself does not need a GPU on this tabular cell, but the same module
command is suitable for RunPod workers so the estimator validation interface is
shared across methods.

Direct estimator usage:

\begin{lstlisting}
from experiments.known_truth import (
    build_known_truth_dgp,
    get_cell,
    known_truth_initial_params,
    simulate_known_truth_panel,
)
from econirl.estimation.nfxp import NFXPEstimator

cell = get_cell("canonical_low_action")
dgp = build_known_truth_dgp(cell.dgp_config)
panel = simulate_known_truth_panel(dgp, cell.simulation_config)

estimator = NFXPEstimator(
    optimizer="BHHH",
    inner_solver="hybrid",
    inner_tol=1e-12,
    inner_max_iter=100_000,
    outer_max_iter=500,
    compute_hessian=True,
    verbose=True,
)

summary = estimator.estimate(
    panel=panel,
    utility=dgp.utility(),
    problem=dgp.problem,
    transitions=dgp.transitions,
    initial_params=known_truth_initial_params(dgp),
)
\end{lstlisting}

\section{Debugging Gaps}

If NFXP fails to recover truth, debug in this order.

\begin{enumerate}
\item \textbf{Feature rank.} If rank is below the number of reward parameters,
$\theta$ is not identified.
\item \textbf{Action support.} If one action is rarely observed, its payoff is
weakly identified.
\item \textbf{Normalization.} If the exit/absorbing anchor is invalid, reward
levels can drift.
\item \textbf{State-only rewards.} The canonical NFXP theta validation needs
action-dependent features.
\item \textbf{Transition tensor.} NFXP needs stochastic transition rows and the
right action-state-next-state orientation.
\item \textbf{Inner tolerance.} Use a tight tolerance such as $10^{-10}$ to
$10^{-12}$ for structural validation.
\item \textbf{High beta.} Use \texttt{inner\_solver="hybrid"} near $\beta=1$;
pure value iteration is intentionally slow there.
\item \textbf{Soft gates.} Do not accept a non-smoke run that only warns.
Known-truth validation should pass hard gates or raise.
\end{enumerate}

\section{Practical Defaults}

Recommended non-smoke validation settings:

\begin{lstlisting}
NFXPEstimator(
    optimizer="BHHH",
    inner_solver="hybrid",
    inner_tol=1e-12,
    inner_max_iter=100_000,
    switch_tol=1e-3,
    outer_tol=1e-6,
    outer_max_iter=500,
    compute_hessian=True,
)
\end{lstlisting}

Use smoke settings only for code-path tests. A smoke test shows that the
estimator runs and returns finite objects; it is not evidence of recovery.
